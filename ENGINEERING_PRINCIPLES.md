# Engineering Principles

OdinSync will be built as a professional open-source platform.

## Principles

1. Business requirements drive architecture.
2. Start with a modular monolith.
3. Evolve to microservices only when justified.
4. Use Domain-Driven Design.
5. Keep bounded contexts clear.
6. Every business entity must belong to a tenant.
7. Never trust tenant_id from the client.
8. Use Flyway for all database changes.
9. Version public APIs.
10. Keep business logic out of controllers.
11. Write tests for important business logic.
12. Documentation is part of the Definition of Done.
13. Use Conventional Commits.
14. Prefer simple solutions before distributed complexity.
