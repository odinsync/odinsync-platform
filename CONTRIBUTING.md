# Contributing

Thank you for your interest in contributing to OdinSync Platform.

## Getting Started

1. Read the README and relevant documentation under `docs/`.
2. Create a focused branch for your change.
3. Keep changes small, reviewed, and tested.
4. Update documentation when behavior, APIs, operations, or architecture change.

## Development Standards

- Prefer clear, maintainable code over clever abstractions.
- Keep tenant isolation, security, and observability in mind for every platform change.
- Record meaningful architecture decisions as ADRs in `docs/adr/`.

## Pull Requests

- Explain the problem and the chosen solution.
- Include tests or explain why tests are not applicable.
- Link related issues or ADRs where relevant.
