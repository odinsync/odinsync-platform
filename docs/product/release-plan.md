# Release Plan

## Pre-Release

- Establish repository structure and documentation.
- Bootstrap backend and infrastructure foundations.
- Define initial domain boundaries.

## MVP

- Ship tenant, identity, customer, and catalog foundations.
- Add basic local development and deployment workflows.
- Establish testing and release standards.

## Post-MVP

- Expand inventory, sales, finance, and notification capabilities.
- Add production operations runbooks.
- Introduce advanced automation and AI capabilities.
